# coze版本党建｜项目说明

> 本文档用于快速说明：这个目录里有哪些版本、分别做什么、应该从哪里开始。

## 1. 项目概览

`coze版本党建` 目录包含两套代码：

- **演示版**（静态展示）
- **生成版**（可持续开发的正式工程）

你可以把它理解为：

- `演示`：偏“看效果”
- `生成`：偏“做产品”

---

## 2. 目录结构（推荐阅读）

```text
coze版本党建/
├── README.md            # 当前说明文档（先看这里）
├── 演示/                # 静态演示版本
│   ├── release/
│   └── preview/
└── 生成/                # 正式工程版本（前后端+部署）
    ├── client/          # 前端（Expo / React Native）
    ├── server/          # 后端（Express + TS）
    └── DELIVERY.md      # 交付与部署说明
```

---

## 3. 两个版本的定位

### 3.1 `演示` 目录（静态演示版）

**存在意义**
- 用于快速展示页面效果和业务原型。

**主要功能**
- 提供可离线查看的静态页面。
- 支持 GitHub Pages 等静态站点托管。

**适用场景**
- 对外展示、评审、汇报。
- 不需要真实后端和数据库联调。

---

### 3.2 `生成` 目录（正式工程版）

**存在意义**
- 用于持续开发和上线交付，承载真实业务逻辑。

**主要功能**
- 前后端完整工程结构。
- 支持接口、权限、数据库接入。
- 包含部署/交付相关配置。

**适用场景**
- 功能迭代、联调测试、生产部署。

---

## 4. 我该看哪个目录？

- **只想看效果 / 做汇报** → 看 `演示`
- **要开发功能 / 接数据库 / 上线** → 看 `生成`

如果你不确定，建议先看 `生成`，因为它是后续可维护的主线版本。

---

## 5. 部署路径建议

- 演示版静态部署根目录：`coze版本党建/演示`
- 生产版前端部署根目录：`coze版本党建/生成/client`
- 生产版后端部署根目录：`coze版本党建/生成/server`

---

## 6. 推荐阅读顺序

### 对产品/管理同学
1. `README.md`（本文件）
2. `演示/release` 或 `演示/preview`

### 对开发同学
1. `README.md`（本文件）
2. `生成/README.md`
3. `生成/client`
4. `生成/server`
5. `生成/DELIVERY.md`

---

## 7. 维护约定（建议）

- 新增目录时，在本 README 的“目录结构”中同步更新。
- 如果版本定位发生变化，优先更新“两个版本的定位”。
- 如果部署方式变化，更新“部署路径建议”和“推荐阅读顺序”。
